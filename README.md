# Выгрузка карточек номенклатуры и контрагентов из УПП 1.3 в БП 3.0

Набор исходников для двух внешних обработок:

- выгрузка из `1С:УПП 1.3` в XML;
- загрузка XML в `1С:Бухгалтерия предприятия 3.0`.

Проект ориентирован на перенос карточек, а не учетных остатков. Остатки товаров и суммы задолженности в БП не создаются и не переносятся.

## Что делает обработка

Выгрузка из УПП 1.3:

- отбирает только номенклатуру, по которой есть остаток;
- отбирает только контрагентов, по которым есть задолженность;
- добавляет родительские группы для сохранения иерархии;
- выгружает карточки номенклатуры и контрагентов в XML;
- выгружает адреса и банковские счета контрагентов;
- выгружает контактных лиц контрагентов;
- выгружает штрихкоды номенклатуры без характеристик и серий.

Загрузка в БП:

- создает или обновляет карточки контрагентов;
- создает или обновляет карточки номенклатуры;
- загружает адреса контрагентов;
- загружает банковские счета контрагентов;
- загружает контактных лиц контрагентов и основное контактное лицо;
- пытается сопоставить или создать единицы измерения;
- использует существующие виды номенклатуры `Товар` и `Услуга` как приоритетные.

Отбор выполняется по итоговому значению по карточке:

- по номенклатуре анализируется остаток;
- по контрагенту анализируется конечная задолженность.

## Сборка выгрузки для УПП 1.3

1. Создать внешнюю обработку на обычных формах.
2. Создать форму `ОсновнаяФорма`.
3. В модуль объекта вставить код из [Обработка.ВыгрузкаКарточекВБП30.МодульОбъекта.bsl](src/upp-export/Обработка.ВыгрузкаКарточекВБП30.МодульОбъекта.bsl).
4. В модуль формы вставить код из [Обработка.ВыгрузкаКарточекВБП30.Форма.ОсновнаяФорма.Модуль.bsl](src/upp-export/Обработка.ВыгрузкаКарточекВБП30.Форма.ОсновнаяФорма.Модуль.bsl).
5. Добавить кнопку `Выгрузить`.
6. Назначить ей обработчик `ВыгрузитьНажатие`.

Минимальная форма сама предлагает выбрать XML-файл и запускает выгрузку с настройками по умолчанию.

## Сборка загрузки для БП 3.0

1. Создать внешнюю обработку.
2. Создать управляемую форму `ОсновнаяФорма`.
3. В модуль объекта вставить код из [Обработка.ЗагрузкаКарточекИзУПП.МодульОбъекта.bsl](src/bp-import/Обработка.ЗагрузкаКарточекИзУПП.МодульОбъекта.bsl).
4. В модуль формы вставить код из [Обработка.ЗагрузкаКарточекИзУПП.Форма.ОсновнаяФорма.Модуль.bsl](src/bp-import/Обработка.ЗагрузкаКарточекИзУПП.Форма.ОсновнаяФорма.Модуль.bsl).
5. При необходимости добавить команду `Загрузить`.
6. Назначить ей обработчик `ЗагрузитьНажатие`.

Минимальный загрузчик предлагает выбрать XML-файл и запускает импорт.

## Настройки по умолчанию

По умолчанию в выгрузке используются:

- регистр остатков `ТоварыНаСкладах`;
- ресурс количества `Количество`;
- регистр взаиморасчетов `ВзаиморасчетыСКонтрагентами`;
- ресурс долга `СуммаВзаиморасчетов`.

При необходимости эти значения можно изменить в коде или адаптировать под конкретную конфигурацию.

Поле `ИмяРегистраВзаиморасчетов` поддерживает список имен через `;` или `,`, если требуется объединять данные из нескольких регистров.

## Что стоит проверить перед использованием

- соответствуют ли имена регистров и ресурсов конкретной базе УПП 1.3;
- подходит ли логика отбора номенклатуры под используемый отчет по остаткам;
- подходит ли логика отбора контрагентов под используемый отчет по взаиморасчетам;
- существуют ли в БП нужные виды номенклатуры и единицы измерения;
- не создаются ли дубли единиц измерения, банков и контрагентов;
- проходят ли тестовую загрузку адреса, банковские счета и контактные лица.

## Ограничения

- переносятся только карточки и связанные базовые реквизиты;
- договоры контрагентов не загружаются;
- остатки и суммы задолженности не загружаются;
- склады не переносятся;
- адреса загружаются по текстовому представлению контактной информации, а при его отсутствии собираются из `Поле1..Поле10`;
- штрихкоды загружаются только без привязки к характеристикам и сериям;
- характеристики, серии и договоры не обрабатываются;
- перед использованием в рабочей базе обмен рекомендуется проверить на копии.
