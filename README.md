# Выгрузка карточек номенклатуры и контрагентов из УПП 1.3 в БП 3.0

Подготовлена заготовка обмена под ваш сценарий:

- из УПП выгружается только номенклатура, у которой есть остаток;
- из УПП выгружаются только контрагенты, у которых есть задолженность;
- остатки и суммы долга в БП не переносятся, только карточки;
- вместе с карточками выгружаются родительские группы.
- для выгруженных контрагентов переносятся адреса и банковские счета;
- для выгруженной номенклатуры переносятся штрихкоды без характеристик и серий;
- при загрузке в БП учитываются обязательные поля `ВидНоменклатуры`, `ЕдиницаИзмерения` и `ЮридическоеФизическоеЛицо`;
- при загрузке виды номенклатуры в БП в первую очередь привязываются к существующим видам `Товар` и `Услуга`;
- если в БП нет нужного вида номенклатуры или единицы измерения, загрузчик пытается создать их автоматически.

Отбор выполняется по итоговому количеству или сумме по самой карточке:

- по номенклатуре суммируется остаток по всем строкам регистра для этой номенклатуры;
- по контрагенту суммируется задолженность по всем строкам регистра для этого контрагента.

## УПП: как собрать выгрузку

1. Создайте внешнюю обработку на обычных формах.
2. Создайте форму `ОсновнаяФорма`.
3. В модуль объекта вставьте код из [Обработка.ВыгрузкаКарточекВБП30.МодульОбъекта.bsl](/e:/DEV/УПП-БП/src/upp-export/Обработка.ВыгрузкаКарточекВБП30.МодульОбъекта.bsl).
4. В модуль формы вставьте код из [Обработка.ВыгрузкаКарточекВБП30.Форма.ОсновнаяФорма.Модуль.bsl](/e:/DEV/УПП-БП/src/upp-export/Обработка.ВыгрузкаКарточекВБП30.Форма.ОсновнаяФорма.Модуль.bsl).
5. Добавьте одну кнопку `Выгрузить`.
6. Назначьте ей обработчик `ВыгрузитьНажатие`.

Это минимальный вариант формы: при нажатии обработка сама предложит выбрать XML-файл, возьмет текущую дату, выгрузит и номенклатуру, и контрагентов, а также использует настройки по умолчанию из модуля объекта.

## БП 3.0: как собрать загрузку

1. Создайте внешнюю обработку.
2. Создайте управляемую форму `ОсновнаяФорма`.
3. В модуль объекта вставьте код из [Обработка.ЗагрузкаКарточекИзУПП.МодульОбъекта.bsl](/e:/DEV/УПП-БП/src/bp-import/Обработка.ЗагрузкаКарточекИзУПП.МодульОбъекта.bsl).
4. В модуль формы вставьте код из [Обработка.ЗагрузкаКарточекИзУПП.Форма.ОсновнаяФорма.Модуль.bsl](/e:/DEV/УПП-БП/src/bp-import/Обработка.ЗагрузкаКарточекИзУПП.Форма.ОсновнаяФорма.Модуль.bsl).
5. Добавьте одну команду `Загрузить`.
6. Назначьте ей обработчик `ЗагрузитьНажатие`.

Это минимальный вариант загрузчика: при открытии формы он сам предложит выбрать XML-файл и сразу запустит загрузку. Команда `Загрузить` остается как повторный ручной запуск. Вид номенклатуры по умолчанию `Товар`, перезапись наименований включена.

## Настройки по умолчанию

В выгрузке сразу заложены имена, подтвержденные вашим отчетом по метаданным УПП 1.3:

- регистр остатков `ТоварыНаСкладах`
- ресурс количества `Количество`
- регистр взаиморасчетов `ВзаиморасчетыСКонтрагентами`
- ресурс долга `СуммаВзаиморасчетов`

Дополнительно в конфигурации есть регистр `СвободныеОстатки`. Его можно использовать вместо `ТоварыНаСкладах`, если вам нужны не все складские остатки, а только свободный остаток без резервов.

Если в вашей УПП названия отличаются из-за доработок, их все равно можно поправить прямо на форме.

Поле `ИмяРегистраВзаиморасчетов` в обработке поддерживает список через `;` или `,`, поэтому при необходимости можно объединять данные сразу из нескольких регистров, но по умолчанию включен только `ВзаиморасчетыСКонтрагентами`.

## Что проверить перед рабочим запуском

- из какого именно регистра в вашей базе нужно брать остатки;
- какой ресурс суммы у регистра взаиморасчетов реально отражает долг;
- есть ли в БП вид номенклатуры `Товар`;
- корректно ли находятся единицы измерения по коду или наименованию;
- нужна ли вам дополнительная загрузка договоров, цен, характеристик или серий.

## Ограничения

- переносятся только карточки и базовые реквизиты;
- договоры контрагентов не создаются;
- остатки и суммы задолженности не загружаются;
- адреса контрагентов загружаются по текстовому представлению контактной информации, а если его нет, собираются из `Поле1..Поле10`, без отдельной реконструкции структуры ФИАС;
- штрихкоды номенклатуры загружаются только те, которые в УПП не привязаны к характеристикам и сериям;
- контактные лица, договоры, характеристики и серии пока не переносятся;
- автоматическое создание единиц измерения в БП выполняется по коду и наименованию из УПП, поэтому на копии базы лучше проверить, что не появляются дубли единиц;
- перед запуском в рабочей базе лучше проверить обмен на копии.
